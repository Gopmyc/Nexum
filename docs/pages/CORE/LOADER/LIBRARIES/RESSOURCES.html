    Overview


    The RESSOURCES library handles management of scripts, external dependencies, file sending, and capability resolution in Nexum.
    It ensures proper client/server inclusion, resolves internal/external dependencies, loads Lua and binary files, and exposes sub-loader environments.

    RESSOURCES:ResolveDependencies


    Resolves all dependencies (internal and external) for a given file or sub-loader.
    Filters based on client/server execution context and reports missing dependencies.

    Parameters

    • table : Dependencies table containing 'INTERNAL' and 'EXTERNAL' keys
    • table : Table defining sides to load (CLIENT, SERVER)
    • table : Optional sub-loader environment for resolving dependencies

    Returns

    • table : Table of resolved scripts keyed by dependency name
    local tDeps = RESSOURCES:ResolveDependencies(tDepsConfig, {CLIENT=true, SERVER=false}, tSubLoader)

    RESSOURCES:GetScript


    Retrieves a script by name from the loaded RESSOURCES table.
    Searches through all groups and returns the first match found.

    Parameters

    • string : Name of the script to retrieve

    Returns

    • table : Script table if found, otherwise nil
    local tScript = RESSOURCES:GetScript("MY_SCRIPT")

    RESSOURCES:IncludeFiles


    Includes a Lua file, a binary module, or executes a function, depending on type and execution side.
    Handles automatic sending of files to clients via AddCSLuaFile when applicable.

    Parameters

    • string|function : File path or function to include
    • table : Sides table {CLIENT = true/false, SERVER = true/false}
    • table : Arguments passed to the file/function
    • table : Sandbox environment
    • boolean : Load as binary module
    • boolean : Load sub-folders if present
    • table : Capabilities table
    • table : Environment profile table

    Returns

    • any : Result of included file or function
    RESSOURCES:IncludeFiles("scripts/init.lua", {CLIENT=true, SERVER=false}, tArgs, tEnv)

    RESSOURCES:LoadBinaryFile


    Loads a binary file as a Lua module using require.
    Useful for decoupled logic and compiled modules.

    Parameters

    • string : Path to the binary file

    Returns

    • any : Loaded module
    local tModule = RESSOURCES:LoadBinaryFile("modules/core.so")

    RESSOURCES:AddCSLuaFile


    Registers a Lua file to be sent to clients.
    Validates existence and file extension. Handles folders recursively.

    Parameters

    • string : Path to Lua file or folder
    RESSOURCES:AddCSLuaFile("client/init.lua")

    RESSOURCES:ResolveCapabilities


    Filters and structures a configuration table based on defined capabilities.
    Supports CLIENT, SERVER, and SHARED sets.
    Returns a read-only table for safe access.

    Parameters

    • table : Base configuration table
    • table : Capabilities table with SHARED, CLIENT, SERVER keys

    Returns

    • table : Read-only table filtered according to capabilities
    local tResolved = RESSOURCES:ResolveCapabilities(tConfig, tCapabilities)

    RESSOURCES:GetSubLoaderEnv


    Creates a sub-loader environment with methods to retrieve the loader, environment, buffer, scripts, and initialization state.
    Uses a callback to provide access to the base libraries.

    Parameters

    • function : Function that returns a libraries table for the sub-loader

    Returns

    • table : Sub-loader environment
    local tEnv = RESSOURCES:GetSubLoaderEnv(function() return tLibs end)
