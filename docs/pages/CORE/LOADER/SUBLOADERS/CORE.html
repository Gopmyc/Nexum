    Overview


    The CORE Sub-Loader handles the initialization and loading of core files for the LOADER system.
    It manages the environment, dependencies, capabilities, and execution of all core scripts.

    SUBLOADER:Initialize


    Initializes the core sub-loader by creating its environment, loading the library references,
    and iterating over the provided content to load each core file into the buffer.

    Parameters

    • table : Content table representing the files to load

    Returns

    • table : Loaded buffer containing the initialized core files
    local coreBuffer = SUBLOADER:Initialize(coreFilesContent)

    SUBLOADER:LoadFile


    Loads a single core file by resolving its dependencies, capabilities, and environment profile,
    then calling the RESOURCES library to include the file.

    Parameters

    • table : File configuration table
    • function|nil : Optional chunk function for reloads

    Returns

    • table : Loaded file object
    • boolean : True if the file is shared (CLIENT-side), false otherwise
    local tFileLoaded, bShared = SUBLOADER:LoadFile(fileData)

    SUBLOADER.__ENV


    Defines the environment of the CORE sub-loader including:
    - ACCESS_POINT: string identifier for this environment
    - CONTENT: table containing a CORE table with its own environment and library references

    print(SUBLOADER.__ENV.ACCESS_POINT) --> "CORE"
