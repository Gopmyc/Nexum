📘 Nexum

    Overview


    The core Relay module of Nexum.
    This module handles module management commands for the Nexum project.
    It supports downloading modules, caching manifests, deleting files, cleaning folders, and executing commands.
    All actions respect CLIENT/SERVER shared state rules.

    CORE:Initialize


    Initializes the relay instance.
    Reads the main MANIFEST file, sets up internal caches, and prepares the folder cleanup map.

    Returns

    • table : Initialized relay instance
    local relay = CORE:Initialize()

    CORE:RunCommand


    Executes a command line via the COMMAND library.

    CORE:CacheManifest


    Caches a module's manifest in the internal `_CACHE.MODULE_MANIFEST`.

    CORE:GetCachedManifest


    Returns a cached manifest table for a given module name.

    CORE:Destroy


    Serializes and saves the MANIFEST file.
    Cleans up all internal state and deletes the relay instance.

    CORE:DeleteFiles


    Deletes all files listed in the cached manifest for a module.
    Cleans empty folders and updates manifest status.

    CORE:CleanEmptyFolders


    Recursively scans directories starting from `sBasePath` and deletes empty folders tracked in `FOLDER_TO_CLEAN`.

    CORE:GetManifestModule


    Fetches the manifest of a module via the NETWORK library.

    CORE:DownloadFile


    Downloads a specific file for a module via NETWORK library.

    CORE:SaveCache


    Saves a string content to the module cache folder.

    CORE:LoadCache


    Loads a module manifest from the cache, parses it, and caches it internally.
