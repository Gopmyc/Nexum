    Overview


    The core Loader module of Nexum.
    This module is responsible for initializing the Loader system, reading and parsing configuration files (YAML), managing libraries and sub-loaders, and providing runtime APIs for instantiation, update, and draw cycles.
    It isolates libraries in controlled environments, manages client/server/shared state, and ensures proper load order based on configuration rules.

    LOADER:Initialize


    Initializes the Loader with given configuration path and libraries table.
    Validates input parameters, loads YAML configurations, sets up logging via MsgC, and prepares the environment for loading libraries and sub-loaders.

    Parameters

    • string : Path to the main configuration file (usually 'configuration/nexum.lua')
    • table : Table of available libraries to load

    Returns

    • table : Initialized loader instance ready to manage libraries and runtime
    local tLibs = {}
local loader = LOADER:Initialize("configuration/nexum.lua", tLibs)

    LOADER:LoadConfiguration


    Recursively loads all YAML configuration files in a given directory.
    Parses each YAML file into Lua tables, merges them according to the configuration structure, and returns a complete configuration table for the loader.
    Handles nested directories and maintains hierarchy.

    Parameters

    • string : Path to the folder containing YAML configuration files

    Returns

    • table : Table representing the merged configuration
    local tConfig = LOADER:LoadConfiguration("configuration/loader")

    LOADER:InitializeSubloaders


    Initializes sub-loaders according to configuration.
    Sub-loaders allow loading specialized groups of modules with defined priorities.
    Ensures load order respects the 'LOAD_PRIORITY' table and integrates with main Loader environment.

    Returns

    • table : Table containing initialized sub-loader instances
    local subLoaders = LOADER:SetupSubLoaders()

    LOADER:GetLibrary


    Retrieves a loaded library by name.
    Ensures proper scope (server/client/shared) and exposes the library API to the caller.
    Performs lazy loading if the library is not already instantiated.

    Parameters

    • string : Name of the library to retrieve

    Returns

    • table : The loaded library instance
    local runtimeLib = LOADER:GetLibrary("runtime")

    LOADER:Update


    Executes the update cycle for all runtime instances managed by the loader.
    Typically called once per frame in the main loop.

    Parameters

    • number : Delta time since last update
    LOADER:Update(dt)

    LOADER:Draw


    Executes the draw/render cycle for all runtime instances managed by the loader.
    Handles rendering or client-side visual updates if applicable.

    LOADER:Draw()

    LOADER:Quit


    Cleans up runtime instances, libraries, and sub-loaders.
    Should be called on application exit to ensure proper resource release.

    LOADER:Quit()
